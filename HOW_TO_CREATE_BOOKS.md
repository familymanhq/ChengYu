## 2. Verify against an online source before drafting

Before writing a single beat, check the README's version of the idiom against at least 2 independent online sources (e.g. chineseidioms.com, cultureyard.net, echineselearning.com, or Wikipedia). Confirm:
- The meaning/translation matches.
- The core story beats (characters, what happens, the twist) match — minor detail differences are fine, but the plot shouldn't contradict.
- The pinyin is correct.

If a source cites the original classical text (e.g. 战国策 *Zhanguo Ce*, 韩非子 *Han Feizi*, 吕氏春秋 *Lüshi Chunqiu*, 管子 *Guanzi*), note it — it's a nice optional credibility footnote even if not used in the book itself.

If something in the README contradicts multiple outside sources, flag it before drafting rather than building a book on a wrong premise.

## 3. Map the README story onto the Story Spine

Every idiom in `README.md` already has a 故事 (story) and English summary — that's your raw material, not something to invent. Stretch it across the 6 fixed beats:

1. **Once there was…** — introduce the character(s) and their normal world.
2. **Every day…** — establish the routine / what's normally true.
3. **Until one day…** — the inciting moment from the README story.
4. **Because of that…** — the action taken / what unfolds.
5. **Until finally…** — the realization or consequence.
6. **And ever since then…** — land on the idiom itself as the payoff line.

Don't force extra plot in — these are single-incident parables, not character arcs. If a beat feels thin, that's fine; the idiom is the point, not the plot.

---

## 4. Write each beat (bilingual)

For every beat, write in this order:
1. **English** — 2–4 sentences, starting with the bolded beat phrase (e.g., **"Once there was"** a musician named...).
2. **Chinese (simplified)** — same content, natural phrasing, not a literal translation.
3. *Pinyin* — italicized, full-sentence transliteration of the Chinese.

### ⚠️ Writing Voice & Style Guidelines (Anti-AI Writing Rules)
Our stories must sound like they were written by a sharp human. Follow these rules strictly:
* **NO Em Dashes**: Never use em dashes (`—` or `--`) in English or Chinese text. Use commas, colons, semicolons, or parentheses instead.
* **Short Phrase-level Lines**: Break down Chinese sentences and Pinyin into short, phrase-level lines in the markdown. This ensures the generator pairs them directly line-by-line, placing Pinyin directly above its corresponding Chinese words on a line basis, preventing misalignments due to text wrapping.
* **NO Negative Parallelisms**: Avoid structures like *"This isn't X. This is Y,"* *"Not X. Y,"* or *"It's not about X, it's about Y."* State what it is directly.
* **Pacing & Rhythm**: Default to short paragraphs (1-2 sentences). Vary sentence length (short punchy lines mixed with longer ones) to break the typical AI "metronome" rhythm.
* **Banned AI Vocabulary**: Never use LLM-fingerprint words like *delve, realm, harness, unlock, tapestry, paradigm, cutting-edge, revolutionize, landscape, intricate, crucial, pivotal, leverage, holistic, vibrant, showcase, etc.*
* **Copulative Avoidance**: Avoid bloated copulative substitutes like *serves as, stands as, represents, marks a, features a*. Just use simple verbs like *is* or *has*.

Defaults established in book #1 (keep unless told otherwise):
- Keep pinyin on every line — it's a language-learning tool, not just decoration.
- Onomatopoeia/sound effects (e.g., *munch, munch, munch* / 嚼，嚼，嚼) are welcome where they add texture — include the Chinese version alongside.

## 5. Lock a Visual Reference block (do this before writing any image prompts)

Right after the title block, add a `## Visual Reference` section with:
- **Style** — one fixed sentence describing medium, palette, and rendering, e.g. *"Soft watercolor illustration in traditional Chinese painting style, warm muted palette of cream, jade green, ochre, and soft red, visible gentle brushstrokes, warm lighting, children's picture book art, no text in image."* Reuse this exact sentence in every prompt in the book.
- **One line per named character** — fixed physical description (hair, clothing, colors, objects they carry/are). Reuse the exact wording every time that character appears in a prompt.

This matters because Gemini has no memory between separate image generations — if you re-describe the musician slightly differently in each prompt, you'll get a different-looking musician in each image. Locking and repeating identical text is the cheapest fix.

**For real continuity (not just close):** generate in one chat thread and feed the previous image back in as a reference ("using this exact character and art style, now draw...") instead of firing off all 6 prompts independently.

---

## 6. Write the 6 image prompts

One 🎨 prompt per beat, placed right after that beat's pinyin line. Each prompt = scene-specific action + the locked character description(s) + the locked style sentence, pasted in verbatim. End every prompt with `no text` (or `no other text` for the closing title-card image) so Gemini doesn't render words into the illustration.

The final beat ("And ever since then…") gets a different *kind* of image — not a story scene, but a title-card: the four characters in bold ink calligraphy, with small cartoon versions of the characters using the same locked descriptions, in the same palette as the rest of the book.

---

## 7. Write the Title Page (curiosity hook)

Add a `## Title Page / 封面` section right after the Visual Reference block (before Beat 1 — "Once there was…"). This is page 1 of the published book: a cover that hooks curiosity before the story starts, using the "curiosity gap" technique — give the reader a real question or mystery statement, not the answer, so they are motivated to keep reading.

Contents, in order:
1. **🎨 One image prompt** — a teaser of the opening scene/character(s), using the locked style sentence + locked character description(s) pasted in verbatim (same rule as every other prompt in the book). It must NOT reveal the twist or ending — show the setup, not the outcome. End with `no text in image.`
2. **Exactly 3 curiosity-gap options**, represented as Markdown/HTML comments in the draft, with **exactly one option uncommented/active by default**. These options **must be custom-tailored specifically to the characters and setup shown in the cover image** (not generic templates):
   - **Option 1: Prediction** (Recommended for age 3–6): Ask what will happen next based on the cover setup (e.g., *"What do you think will happen when the musician plays for the cow? / 当音乐家给牛弹琴时，你觉得会发生什么呢？"*).
   - **Option 2: Mystery** (Creates intrigue): Describe the cover situation and invite speculation (e.g., *"A musician is walking toward a grazing cow with his zither... What is about to happen? / 一个音乐家抱着琴朝吃草的牛走去……接下来会发生什么奇怪的事呢？"*).
   - **Option 3: Wisdom Discovery** (Reflective): Ask about the lesson behind the setup (e.g., *"What lesson can we learn from a musician and a cow? / 从音乐家和牛的故事里，我们能学到什么道理呢？"*).
3. **A closing nudge**, bilingual: `Keep reading to find out! / 继续往下读吧！`

See `Books/01_dui_niu_tan_qin.md` for the worked example.

This step comes *after* Steps 4–6 (beats + visual reference + image prompts), since it draws on the locked character descriptions for the teaser image.
